# コーディングテスト対策

# 前半
## 流れ
- 最初にコードの方針を立てる。
  - 問題文の読み込み
  - 流れの部分だけのコードを書く。
- 実装。
- コードの計算量の見積もり。

## 全体的な注意点
- 説明はハイレベルな結論を述べて、具体的な説明に入る。   
- 実装しながら、ダラダラ喋らない。喋るときは喋る、実装するときは実装する。
- 今何の話をしているのかがわかるようにする。  

## 方針建ての注意点
- 具体的な実装の説明(配列にpush_back)はせず、ロジックだけを説明することを意識

## 面接官がチェックしたい能力
- 今、問題をどのように捉えていて、それに対して何を考えていて、どんなことをしたいのかを伝えることができる能力を持っているのか。(物事を筋道立てて解決する能力と、その立てた筋道や思考をちゃんと伝えられる能力)  
- 細かいニュアンスはだいぶ拾ってくれるよ。  

# 後半

## 流れ
できたコードに対して質問や指摘をされたり、テストケースを考えましょうとか、そういうディスカッションが入ってくる。
